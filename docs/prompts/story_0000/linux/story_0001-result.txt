● できます。mise が最適解で、実際にこのプロジェクトのWSL2側ツールはほぼ全部 mise
  で固定できることを確認・設定しました。

  なぜ mise が向くか

  - ディレクトリ単位で版を切替：このリポジトリに入ると mise.toml
  の版に自動で切り替わり、他プロジェクトのグローバル Python/pip/Go
  を汚さない（「版の相違」問題がそのまま解決）。
  - pip を直接使わない設計：Python 依存は各サービスの uv sync（既存の ci_python.sh
  どおり）でサービス毎の隔離 venv に入るので、pip版の差異の影響を受けません。mise は「どの python/uv
  を使うか」を固定するだけ。

  設定したもの（mise.toml）

  ┌──────────────┬────────────────────────────┬─────────────────────────────────────────────────────┐
  │    ツール    │         mise 指定          │                        備考                         │
  ├──────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ go           │ go = "1.22"                │ auth/api                                            │
  ├──────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ python       │ python = "3.11"            │ precompiled（python.compile=falseでビルド不要・高速 │
  │              │                            │ ）                                                  │
  ├──────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ uv           │ uv                         │ Python依存管理                                      │
  ├──────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤

  │ golangci-lin │ golangci-lint              │ Go Lint                                             │

  │              │ e                          │                                                     │
  ├──────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ pre-commit   │ pipx:pre-commit            │                                                     │

  - [settings] lockfile = true → mise install 時に mise.lock
  が生成され、チーム間で完全再現。コミット推奨。
